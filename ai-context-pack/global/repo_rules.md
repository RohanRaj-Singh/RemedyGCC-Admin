# Repository Rules

## File Creation Rules

- ALWAYS create files inside the CURRENT working directory
- NEVER create files outside the project unless explicitly told
- NEVER create parent-level folders unnecessarily

## Folder Discipline

Follow existing structure:

/apps
/services
/ai-context

If unsure:

👉 ask before creating new structure

## Naming Rules

- use kebab-case for folders
- use camelCase for variables
- use PascalCase for components

## Component Rules

- keep components small and reusable
- avoid duplication
