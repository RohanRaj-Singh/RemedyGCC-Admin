# Git Rules

## Every Change Must:

- be meaningful
- be grouped logically

## Commit Style

Use:

type(scope): message

Examples:

feat(admin): add tenant table
fix(scanner): correct weight calculation
refactor(ui): simplify dashboard layout

## Before Finishing Task

ALWAYS:

1. Ensure code compiles
2. Ensure no obvious errors
3. Suggest commit message

## Branching (optional)

feature/super-admin-ui
feature/scanner-builder
